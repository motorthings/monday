Architecture complete. A request enters via the intake form, gets AI-summarized, formula-checked for completeness, routed to the Portfolio Board for PM review, approved, and auto-linked to a new Execution Board item. Status rolls up through mirror columns. The executive dashboard shows it all live. Zero manual spreadsheet-to-PowerPoint work for Pepper.

What rolls up vs. what stays local: Status and Due Date roll up (mirror columns). Sprint assignments and daily task details stay local to the Execution Board. This separation is deliberate — executives see trajectory, not micro-detail.
